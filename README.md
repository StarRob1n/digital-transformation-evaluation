制造业企业数字化转型评价方法研究

本项目是我（裴翊成）本科毕业论文的完整代码实现与数据模板。  
论文针对制造业数字化转型评价中的 静态权重、阶段-权重循环依赖 等问题，提出了一种基于 改进熵权-TOPSIS 的动态评价方法，并提供了企业可自行使用的验证工具。



📌 主要特点

- ✅ 基于国家标准 GB/T 39116-2020 构建 4维度 × 36项指标 评价体系  
- ✅ 提出 “关键门槛指标法” 独立判定企业转型阶段（L1～L5），打破循环逻辑  
- ✅ 实现 动态维度权重调整，不同阶段企业采用差异化评价重心  
- ✅ 内置 6家行业参考企业矩阵（覆盖 L1～L5），为熵权法提供稳定基准  
- ✅ 完成 等权重对比 + 三方案对比 + 蒙特卡洛敏感性分析（500次扰动）  
- ✅ 提供 企业自评 Excel 模板，填写后一键输出评价结果（阶段 / 得分 / 短板 / 建议）

🚀 快速开始

1. 克隆仓库

bash
git clone https://github.com/StarRob1n/digital-transformation-evaluation.git
cd digital-transformation-evaluation


2. 安装依赖

bash
pip install numpy pandas openpyxl


3. 运行论文模型（复现论文结果）

bash
python 03_评价模型.py


输出：`模型结果.xlsx`（包含 4 个工作表）

4. 运行企业自评（使用自己的数据）

- 打开 `企业验证数据模板.xlsx`，在「指标填写表」的 **K 列** 填入贵企业 36 项指标值  
- 运行校验（推荐）：

bash
python 02_数据逻辑校验.py


- 运行验证：

bash
python 03_运行企业验证.py


按提示输入企业名称、行业，即可获得 `验证结果.xlsx`。



📊 模型核心逻辑


原始数据（36项指标）
    ↓
门槛指标法 → 转型阶段（L1～L5）
    ↓
动态权重调整（熵权基础权重 + 阶段调节系数）
    ↓
加权 TOPSIS → 综合得分 + 分维度得分
    ↓
短板识别 + 优化建议


稳健性验证：  
- 三方案对比（动态权重 vs 传统熵权 vs 等权重）  
- 单指标权重 ±10% 扰动  
- 蒙特卡洛 500 次组合扰动 → 95% 置信区间宽度均 < 1 分



📄 论文摘要

制造业数字化转型评价中，现有方法存在静态权重、阶段‑权重循环依赖、样本依赖三大局限。本文基于 GB/T 39116-2020，构建了包含技术基础、业务应用、组织管理、转型成效 4 个维度共 36 项指标的评价体系，提出“关键门槛指标法”实现转型阶段的前置独立判定，并通过动态权重调整机制使同一指标在不同阶段获得差异化权重。采用熵权‑TOPSIS 法计算综合得分，以 3 家仿真企业和 6 家行业参考企业为样本进行验证。蒙特卡洛敏感性分析表明模型具有较好的稳健性，等权重对比实验证明动态权重能够更准确地揭示企业短板维度，为分类施策提供了量化依据。

关键词：制造业数字化转型；成熟度评价；熵权‑TOPSIS；动态权重；门槛指标法
